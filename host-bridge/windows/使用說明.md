# Windows Host Bridge 使用說明

本資料夾是同一個 GPT Relay repo 的 Windows host 端工具。它讓 Linux VM、WSL、Docker 或另一台機器中的 Codex，控制 Windows 上已登入 ChatGPT 的 Chrome。

```text
Linux VM / Codex + GPT Relay plugin
          | HTTP :8765 + Bearer Token
Windows host: host-bridge -> Chrome CDP :9222 -> ChatGPT
```

## 何時需要這組工具

- Codex 和已登入 ChatGPT 的 Chrome 在不同環境：需要。
- Codex 可以直接控制同一環境內的 Chrome：通常不需要；直接使用 GPT Relay plugin 即可。

完整模式比較看 [部署模式](../../docs/deployment-modes.md)。Linux VM 對 Windows host 的逐步設定看 [快速開始](../../user_quick_start.md)。

## Windows 第一次設定

1. 在 Windows 上取得這個 repo，保留 `host-bridge/` 完整資料夾。
2. 安裝 Google Chrome 與 Node.js，確認 `node -v`、`npm -v` 有結果。
3. 打開 [`gpt-relay.ps1`](./gpt-relay.ps1)，在 CONFIG 區更改 Token；跨機器時不要使用預設 `change-me`。
4. 雙擊 `setup-host-bridge-deps.bat`，或直接雙擊 `run-gpt-relay-host.bat`，後者會在缺依賴時自動安裝。

## 日常操作

- `run-gpt-relay-host.bat`：關閉目前所有 Chrome、開啟專用 debug Chrome、啟動 bridge 並驗證 `/health`。
- `status-gpt-relay-host.bat`：檢查 Chrome `9222`、bridge `8765` 和 Token 驗證。
- `stop-gpt-relay-host.bat`：停止 bridge，並只關閉 GPT Relay debug Chrome。

`run` 會關閉所有 `chrome.exe`；先儲存一般 Chrome 中尚未保存的工作。啟動後，請在新開的 debug Chrome 登入 ChatGPT，並保持 bridge 的命令列視窗開啟。

## Linux VM / Codex 設定

把 `<windows-host-ip>` 換成 Windows host 的可達 IP，Token 要和 Windows CONFIG 一致：

```bash
export GPT_RELAY_BROWSER_PROVIDER=host-bridge
export GPT_RELAY_HOST_BRIDGE_URL=http://<windows-host-ip>:8765
export GPT_RELAY_HOST_BRIDGE_TOKEN='<your-token>'
```

連線測試：

```bash
curl -H 'Authorization: Bearer <your-token>' \
  http://<windows-host-ip>:8765/health
```

成功後，在 Codex 選取 `GPT Relay` plugin，輸入要交給 ChatGPT 的任務。

## 常見問題

- `9222` 失敗：Chrome 路徑不正確、debug Chrome 未啟動，或 port 被占用。
- `8765`／`/health` 失敗：bridge 沒起來、Token 不一致，或 bridge 無法連到 Chrome。
- Linux VM 連不到：Windows firewall 未允許 `8765`、bridge 未綁定 `0.0.0.0`，或使用了錯誤 host IP。
- 跨機器請設定強 Token，優先使用 SSH tunnel、Tailscale 或受控內網；不要暴露 Chrome CDP `9222`。
